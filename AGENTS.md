# Instructions for Claude Code

## Project overview

Odoo development environment using:
- **mise** for toolchain management (Python 3.12, Node.js)
- **Git worktrees** to work on multiple branches simultaneously
- **Bare repositories** centralized in `.cache/git/`

## Directory structure

```
odoo-env/
├── .cache/                   # Gitignored runtime data
│   ├── git/                  # Bare repositories
│   │   ├── community.git     # github.com/odoo/odoo
│   │   └── enterprise.git
│   ├── postgres-data/        # Postgres volume (docker)
│   └── mailpit-data/         # Mailpit volume (docker)
├── workspaces/{name}/        # Gitignored generated workspace files
│   ├── odoorc                # Merged Odoo config (generated by oo)
│   ├── {name}.code-workspace # VS Code workspace
│   └── zed/
│       ├── .zed/settings.json, tasks.json, launch.json
│       ├── community, enterprise, … → symlinks to src/
│       ├── odoo-env → symlink to repo root
│       └── odools.toml → symlink to repo root/odools.toml
├── src/                      # Worktrees (gitignored)
│   ├── community/{branch}/
│   ├── enterprise/{branch}/
│   └── ...
├── scripts/                  # Auto-added to $PATH by mise
│   ├── oo                    # Entry point (calls oo package)
│   ├── odoo-shell-aliases.sh  # Shell functions (bash/zsh) — source manually
│   ├── odoo-shell-aliases.fish # Shell functions (fish) — source manually
│   └── install-wkhtmltopdf  # Debian/Ubuntu only, kept as bash
├── oo/                       # Python package (main logic)
├── .venv/                    # Python venv (managed by mise)
├── compose.yml               # Optional dev services
├── mise.toml
├── odoo-env.toml             # Profile declarations (gitignored, copy from .example)
├── odoo-env.toml.example     # Committed template
├── odools.toml               # Odoo Language Server config (gitignored, auto-generated)
└── .gitignore
```

## Scripts

### `oo/` package + `scripts/oo` entry point

Main entry point. Reads `odoo-env.toml` (`[[profile]]` sections), ensures bare repos and
worktrees exist, then either launches a subshell or performs the requested operation.
If a bare repo is missing, it is cloned automatically. Also regenerates `odools.toml`
and workspace files on each `activate` run.

**Subcommands:**

```bash
oo list                          # List all profiles from odoo-env.toml
oo activate <profile>            # Set up worktrees and launch a shell
oo <profile>                     # Shorthand for activate
oo status [profile]              # Show git status (ahead/behind/dirty) for all worktrees
oo rebase [profile]              # Fetch + rebase worktrees onto upstream
oo remove <profile>              # Remove worktrees for a profile
oo gen [--ide vscode|zed|all] <profile>  # Regenerate IDE workspace files only
```

Omitting `[profile]` for `status` and `rebase` targets the current active profile
(via `$ODOO_ENV_NAME`) when inside a shell, or all profiles otherwise.

### `scripts/odoo-shell-aliases.sh` / `odoo-shell-aliases.fish`

Shell functions for daily Odoo dev work. Add to your shell rc file:

```bash
# bash/zsh (~/.bashrc or ~/.zshrc)
source /path/to/odoo-env/scripts/odoo-shell-aliases.sh

# fish (~/.config/fish/config.fish)
source /path/to/odoo-env/scripts/odoo-shell-aliases.fish
```

| Function | Action |
|---|---|
| `oo [args]` | Start Odoo (`odoo-bin -c $ODOO_RC`) |
| `otest [args]` | Run tests (`--test-enable --stop-after-init`) |
| `ocd` | `cd $ODOO_PATH` |
| `ocd-e` | `cd $ODOO_ENTERPRISE_PATH` |
| `ogit [args]` | `git -C $ODOO_PATH` |
| `olog` | `tail -f` the configured logfile |
| `odb [dbname]` | `psql` with DB settings from `$ODOO_RC` |
| `oenv` | Print current environment summary |


## Dev services (optional)

`compose.yml` provides Postgres+pgvector, mailpit, and pgweb. Never started
automatically — opt-in only.

```bash
mise run services       # start postgres, mailpit, pgweb
mise run services-down  # stop all
```

- **Postgres** (pgvector/pgvector:pg17) → `localhost:5432`, user/pass: `odoo/odoo`
- **Mailpit** → UI `localhost:8025`, SMTP `localhost:1025`
- **pgweb** → `localhost:8081` (browser DB client)

## Odoo configuration

All profiles are declared in `odoo-env.toml` (gitignored). Copy from
`odoo-env.toml.example` to get started:

```bash
cp odoo-env.toml.example odoo-env.toml
```

`[odoorc]` holds global Odoo conf defaults (DB host, SMTP, etc.).
`odoorc.X = ...` inside a `[[profile]]` holds per-profile overrides (port, db_name, etc.).
`[remote.X]` declares custom repo URLs for non-odoo repositories.

`oo` merges `[odoorc]` + per-profile `odoorc.*`, injects `addons_path` and
`http_port`, and writes the result to `workspaces/{name}/odoorc`.
`ODOO_RC` is set to that path automatically.

## Common commands

`scripts/` is automatically added to `$PATH` by mise when inside the repo
(requires mise shell activation — see README). Use scripts directly by name.

```bash
# List profiles declared in odoo-env.toml
oo list

# Create worktrees and launch shell
oo my-main

# Regenerate IDE workspace files only (no shell)
oo gen my-main
oo gen --ide vscode my-main
oo gen --ide zed my-main

# Install Python deps for a version
pip install -r src/community/master/requirements.txt

# Install rtlcss
npm install -g rtlcss
```

## Environment variables set by oo

| Variable               | Description                                  |
|------------------------|----------------------------------------------|
| `ODOO_VERSION`         | Base branch (e.g. `master`, `18.0`)          |
| `ODOO_RC`              | Path to `workspaces/{name}/odoorc`           |
| `ODOO_PATH`            | Path to community worktree                   |
| `ODOO_ENTERPRISE_PATH` | Path to enterprise worktree (empty if none)  |
| `ODOO_EXTRA_PATHS`     | Colon-separated paths of custom repos        |
| `ODOO_ADDONS_PATH`     | Comma-separated addons paths                 |
| `PYTHONPATH`           | Includes community worktree root             |
| `ODOO_PORT`            | Port (default: 8069)                         |

## Notes

- `src/`, `.cache/`, `workspaces/`, `odoo-env.toml`, `odools.toml` are gitignored — never commit them
- `.notes/` is gitignored except `.gitkeep`; same pattern for `src/` with `src/.gitkeep`
- `.claude/` is gitignored (local Claude Code settings)
- The Python venv at `.venv/` is managed by mise automatically
- `install-wkhtmltopdf` is intentionally kept as bash (Debian/apt-specific)
- Worktrees are created on a local tracking branch (not detached HEAD)
- `odools.toml` is auto-generated on each `oo <profile>` run for the Odoo LSP
- `workspaces/{name}/zed/odools.toml` is a symlink to the repo root `odools.toml`

## Custom repos

To use non-odoo repositories, declare their URL in `[remote.X]` and reference the branch
in the `[[profile]]`:

```toml
[remote.my-addons]
origin = "git@github.com:myclient/odoo-addons.git"

[[profile]]
name = "my-client"
branch.community = "17.0"
branch.enterprise = "17.0"
branch.my-addons = "main"
```

Standard Odoo repos (`community`, `enterprise`, `design-themes`, `industry`, `upgrade`, `util`) do not need a URL — they default to `git@github.com:odoo/{alias}.git`.

## odools.toml reference (pre-generation format)

```toml
[[config]]
name = "Odoo Dev"
odoo_path = "/abs/path/src/community/master"
addons_paths = [
    "/abs/path/src/community/master/addons",
    "/abs/path/src/community/master/odoo/addons",
    "/abs/path/src/enterprise/master",
]
python_path = "/abs/path/.venv/bin/python3"
```

VSCode users can also point to an alternative config file:
`"Odoo.serverConfigPath": "/path/to/odoo-env/odools.toml"`

The symlink at `workspaces/{name}/zed/odools.toml` allows Zed to find `odools.toml`
automatically when the workspace is opened from that directory.
