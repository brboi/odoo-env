# Instructions for Claude Code

## Project overview

Odoo development environment using:
- **mise** for toolchain management (Python 3.12, Node.js)
- **Git worktrees** to work on multiple branches simultaneously
- **Bare repositories** centralized in `.cache/git/`

## Directory structure

```
odoo-env/
├── .cache/                   # Gitignored runtime data
│   ├── git/                  # Bare repositories
│   │   ├── community.git     # github.com/odoo/odoo
│   │   └── enterprise.git
│   ├── postgres-data/        # Postgres volume (docker)
│   ├── mailpit-data/         # Mailpit volume (docker)
│   └── odoo-active.conf      # Generated by odoo-env (merged config)
├── src/                      # Worktrees (gitignored)
│   ├── community/{branch}/
│   ├── enterprise/{branch}/
│   └── ...
├── config/
│   ├── odoo.conf             # Base Odoo config (committed)
│   ├── odoo.local.conf       # Local overrides (gitignored)
│   └── odoo.local.conf.example
├── scripts/                  # Auto-added to $PATH by mise
│   ├── odoo-env              # Main script (Python)
│   ├── setup-odoo-repository # Bare repo init (Python)
│   └── install-wkhtmltopdf  # Debian/Ubuntu only, kept as bash
├── .venv/                    # Python venv (managed by mise)
├── compose.yml               # Optional dev services
├── mise.toml
├── odools.toml               # Odoo Language Server config
└── .gitignore
```

## Scripts

### `scripts/odoo-env`

Main entry point. Parses a version spec, ensures bare repos and worktrees exist, then either launches a subshell or prints env vars.

**Version spec format:**
```
BRANCH
BRANCH:repo#BRANCH
BRANCH:repo#BRANCH:repo2#BRANCH2
```

**Options:** `--port`, `--setup-only`, `--print-env`

### `scripts/setup-odoo-repository`

Initializes a bare repo in `.cache/git/{alias}.git`. The only hardcoded alias: `odoo` → `community` (mirrors `github.com/odoo/odoo` as `community` locally). All other repo names are used as-is.

Configures:
- `origin` → `git@github.com:odoo/{repo}.git`
- Refspecs for branches and pull requests (`origin/pr/*`)
- `dev` remote → `git@github.com:odoo-dev/{repo}.git` (detected via `git ls-remote`, only if it exists)


## Dev services (optional)

`compose.yml` provides Postgres+pgvector, mailpit, and pgweb. Never started
automatically — opt-in only.

```bash
mise run services       # start postgres, mailpit, pgweb
mise run services-down  # stop all
```

- **Postgres** (pgvector/pgvector:pg17) → `localhost:5432`, user/pass: `odoo/odoo`
- **Mailpit** → UI `localhost:8025`, SMTP `localhost:1025`
- **pgweb** → `localhost:8081` (browser DB client)

Devs with their own Postgres: just skip `mise run services` and override DB
settings in `config/odoo.local.conf`.

## Odoo configuration

`config/odoo.conf` (committed) holds base settings pointing to the compose stack.
Create `config/odoo.local.conf` (gitignored) for local overrides. See
`config/odoo.local.conf.example` for available options.

`odoo-env` merges both files at startup, injects `addons_path`, and writes the
result to `.cache/odoo-active.conf`. `ODOO_RC` is set to that path automatically.

## Common commands

`scripts/` is automatically added to `$PATH` by mise when inside the repo
(requires mise shell activation — see README). Use scripts directly by name.

```bash
# Set up bare repos ('odoo' is automatically aliased to 'community')
setup-odoo-repository odoo
setup-odoo-repository enterprise

# Create worktrees and launch shell
odoo-env master
odoo-env 18.0:enterprise#my-feature

# Setup only (no shell)
odoo-env master --setup-only

# Print env vars (e.g. for eval)
eval "$(odoo-env master --print-env)"

# Install Python deps for a version
pip install -r src/community/master/requirements.txt

# Install rtlcss
npm install -g rtlcss
```

## Environment variables set by odoo-env

| Variable               | Description                                  |
|------------------------|----------------------------------------------|
| `ODOO_VERSION`         | Base branch (e.g. `master`, `18.0`)          |
| `ODOO_RC`              | Path to `.cache/odoo-active.conf`            |
| `ODOO_PATH`            | Path to community worktree                   |
| `ODOO_ENTERPRISE_PATH` | Path to enterprise worktree                  |
| `ODOO_ADDONS_PATH`     | Comma-separated addons paths                 |
| `PYTHONPATH`           | Includes community worktree root             |
| `ODOO_PORT`            | Port (default: 8069)                         |

## Notes

- `src/`, `.cache/`, `config/odoo.local.conf` are gitignored — never commit them
- `.personal-notes/` is gitignored except `.gitkeep`
- `.claude/` is gitignored (local Claude Code settings)
- The Python venv at `.venv/` is managed by mise automatically
- `install-wkhtmltopdf` is intentionally kept as bash (Debian/apt-specific)
- Worktrees are created in detached HEAD state from `origin/{branch}`
