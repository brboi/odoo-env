# Instructions for Claude Code

## Project overview

Odoo development environment using:
- **mise** for toolchain management (Python 3.12, Node.js)
- **Git worktrees** to work on multiple branches simultaneously
- **Bare repositories** centralized in `.cache/git/`

## Directory structure

```
odoo-env/
├── .cache/                   # Gitignored runtime data
│   ├── git/                  # Bare repositories
│   │   ├── community.git     # github.com/odoo/odoo
│   │   └── enterprise.git
│   ├── postgres-data/        # Postgres volume (docker)
│   ├── mailpit-data/         # Mailpit volume (docker)
│   └── envs/{name}/odoorc    # Generated by odoo-env (merged config)
├── src/                      # Worktrees (gitignored)
│   ├── community/{branch}/
│   ├── enterprise/{branch}/
│   └── ...
├── scripts/                  # Auto-added to $PATH by mise
│   ├── odoo-env              # Main script (Python)
│   └── install-wkhtmltopdf  # Debian/Ubuntu only, kept as bash
├── .venv/                    # Python venv (managed by mise)
├── compose.yml               # Optional dev services
├── mise.toml
├── odoo-env.toml             # Environment declarations (gitignored, copy from .example)
├── odoo-env.toml.example     # Committed template
├── odools.toml               # Odoo Language Server config
└── .gitignore
```

## Scripts

### `scripts/odoo-env`

Main entry point. Reads `odoo-env.toml`, ensures bare repos and worktrees exist, then either launches a subshell or prints env vars. If a bare repo is missing, it is cloned automatically (no need to run a separate setup script).

**Options:** `--list`, `--port`, `--setup-only`, `--odoorc`, `--print-env`

`--odoorc` regenerates `.cache/envs/{name}/odoorc` from current `odoo-env.toml` settings without fetching, creating worktrees, or launching a shell. Useful after editing port or odoorc settings.


## Dev services (optional)

`compose.yml` provides Postgres+pgvector, mailpit, and pgweb. Never started
automatically — opt-in only.

```bash
mise run services       # start postgres, mailpit, pgweb
mise run services-down  # stop all
```

- **Postgres** (pgvector/pgvector:pg17) → `localhost:5432`, user/pass: `odoo/odoo`
- **Mailpit** → UI `localhost:8025`, SMTP `localhost:1025`
- **pgweb** → `localhost:8081` (browser DB client)

## Odoo configuration

All environments are declared in `odoo-env.toml` (gitignored). Copy from
`odoo-env.toml.example` to get started:

```bash
cp odoo-env.toml.example odoo-env.toml
```

`[_.odoorc]` holds global Odoo conf defaults (DB host, SMTP, etc.).
`[env.odoorc]` holds per-environment overrides (port, db_name, etc.).
`[_.branches]` holds global branch defaults (useful for shared repos like `upgrade`).

`odoo-env` merges `[_.odoorc]` + `[env.odoorc]`, injects `addons_path` and
`http_port`, and writes the result to `.cache/sessions/{slug}/odoo.conf`.
`ODOO_RC` is set to that path automatically.

## Common commands

`scripts/` is automatically added to `$PATH` by mise when inside the repo
(requires mise shell activation — see README). Use scripts directly by name.

```bash
# List environments declared in odoo-env.toml
odoo-env --list

# Create worktrees and launch shell
odoo-env my-main

# Setup only (no shell)
odoo-env my-main --setup-only

# Print env vars (e.g. for eval)
eval "$(odoo-env my-main --print-env)"

# Install Python deps for a version
pip install -r src/community/master/requirements.txt

# Install rtlcss
npm install -g rtlcss
```

## Environment variables set by odoo-env

| Variable               | Description                                  |
|------------------------|----------------------------------------------|
| `ODOO_VERSION`         | Base branch (e.g. `master`, `18.0`)          |
| `ODOO_RC`              | Path to `.cache/envs/{name}/odoorc`          |
| `ODOO_PATH`            | Path to community worktree                   |
| `ODOO_ENTERPRISE_PATH` | Path to enterprise worktree                  |
| `ODOO_ADDONS_PATH`     | Comma-separated addons paths                 |
| `PYTHONPATH`           | Includes community worktree root             |
| `ODOO_PORT`            | Port (default: 8069)                         |

## Notes

- `src/`, `.cache/`, `odoo-env.toml` are gitignored — never commit them
- `.notes/` is gitignored except `.gitkeep`
- `.claude/` is gitignored (local Claude Code settings)
- The Python venv at `.venv/` is managed by mise automatically
- `install-wkhtmltopdf` is intentionally kept as bash (Debian/apt-specific)
- Worktrees are created in detached HEAD state from `origin/{branch}`
